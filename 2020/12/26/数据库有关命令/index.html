1. 创建数据库

CREATE DATABASE 或者 CREATE SCHEMA
Create datavabse szxy;
创建数据库 并指定数据库中数据的编码
CREATE DATABASE szxy CHARACTER SET utf8

2. 查看数据库的有关命令

A: 查看数据库MySQL服务器中的所有的数据库:
show databases;
B:查看某个数据库的定义的信息:
show create database 数据库名;
例如：
show create database szxy;

C: 删除数据库
drop database 数据库名称;
例如：
drop database szxy;
D: 修改数据库
ALTER DATABASE [字符集名称a] character set [字符集名称b]

3. 其他的数据库操作命令

A:切换数据库：
use 数据库名;
例如：
Use zhouyu;

B:查看正在使用的数据库:
select database();

二 .表的有关命令

1：创建表
A: 约束的作用:
create table 表名(
       列名 类型(长度) 约束,
       列名 类型(长度) 约束
    );
    限制每一列能写什么数据,不能写什么数据。

CREATE TABLE student (
sno varchar(10) COLLATE utf8_bin NOT NULL COMMENT ‘学号’,
sname varchar(20) COLLATE utf8_bin NOT NULL COMMENT ‘姓名’,
ssex char(2) COLLATE utf8_bin NOT NULL DEFAULT ‘男’ COMMENT ‘性别’,
sbirth date NOT NULL COMMENT ‘出生日期’,
zno varchar(4) COLLATE utf8_bin NOT NULL COMMENT ‘专业号’,
sclass varchar(10) COLLATE utf8_bin NOT NULL COMMENT ‘班级’,
PRIMARY KEY (sno),
KEY zno (zno),
CONSTRAINT zno FOREIGN KEY (zno) REFERENCES specialty (zno)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COLLATE=utf8_bin;
注解：col_name:字段名
Type: 声明字段的数据类型
NOT NULL 或者NULL：表示字段是否可以为空
Default :指定字段的默认值
AUTO_INCREMENT:设置自增属性
RIMARY KEY：对字段指定唯一主键约束
UNiQue KEY：对字段的唯一性约束

B: 哪些约束:

 数据完整性约束（实体完整性，参照完整性，域完整性，用户自定义的完整性）

字段约束 （1.外键约束，2.主键约束，3.非空约束，4.唯一约束，5默认约束，6自增约束，7检查约束

C 查看表的名字

命令: Show TABLE
查看表的结构
命令：show create table

D 修改数据表

1：修改数据表名
ALTER TABLE +旧表名+RENAME TO+新表名；
2：修改字段数据类型
命令：ALTER TABLE+表名+MODIFY+字段名称+数据类型；
例子：ALTER TABLE student MODIFY sname VARCHAR (30)；
3:修改字段名称
命令：ALTER　TABLE+表名+CHANGE+旧字段名+新字段名+新数据类型；
例子：ALTER　TABLE student CHANGE sbirth sdata date;；//将student表的sbirth字段名变为sdata,字段类型为 data
4:添加数据表字段；
命令：ALTER TABLE +表名+ ADD +新字段名+数据类型+约束条件
5：删除字段
命令：ALTER TABLE +表名+DROP+字段名；
6：修改字段排序
命令：ALETER TABLE +表名+ MODIFY +字段名1+数据类型+FIRST AFTER+字段2//FIRST 表示将字段1修改为表的第一个字段 AFTER 表示将字段1插入到字段2中。
7：更改表的存储引擎
命令：ALETER TABLE student ENGINE=MySIM;
8:删除表的外键约束
命令：ALTER TABLE student DROP FOREING KEY zno;

E 对表的操作

1：复制数据表
命令:CREATE TABLE student LIKE stu;
2: 删除数据表
命令：DROP TABLE +表名；
